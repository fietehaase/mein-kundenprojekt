# Dokumentation

Dieses Projekt nutzt eine schlanke Solo-Variante der Modus-Operandi-Methodik.

## Aktive Artefakte

- `spec.md` - fachliche Single Source of Truth; ersetzt `prd.md`.
- `architecture.md` - technischer Rahmen, Datenmodell, Struktur und Standards.
- `decisions.md` - chronologisches Log für Architektur- und Produktentscheidungen.
- `modus-operandi.md` - projektspezifische Arbeitsweise.
- `concepts/` - optionale Feature-Konzepte vor größeren Implementierungen.
- `audit/` - optionale Codebase-Audits.

## Bewusst nicht angelegt

- `prd.md` - wird durch `spec.md` ersetzt.
- `meetings/` - nicht nötig für dieses Solo-Projekt.
- `results/` - nicht nötig für dieses Solo-Projekt.
- `INBOX.md` - erst sinnvoll bei parallelen Doc-Edits oder mehreren Worktrees.
- `backlog.md` - erst einführen, wenn stabile Feature-IDs gebraucht werden.
